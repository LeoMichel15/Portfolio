Vote

Contexte

Sujet : Création d'une application Web qui permet de voter des propositions à une question.


Objectif : Créer une application web de vote en ligne avec de multiples fonctionnalités utilisateur comme des différents rôles : Admin, Utilisateur, Organisateur etc. Par ailleurs, le site web doit posséder sa propre base de données pour gérer les différents utilisateurs.


Code existant : Il n’y avait pas de code donné, nous sommes partis de zéro.


Base de données : Le projet utilise PostgreSQL pour administrer la base de données. L'objectif n'était pas de l’optimiser, mais d’avoir une bd fonctionnelle.


Le but du projet est donc de créer une application en PHP, HTML et CSS qui permet à l'utilisateur de voter sur des questions et de pouvoir potentiellement donner son avis. Ce projet s'apparente à la création d’un forum.

Analyse

Lors de ce projet, j'ai pu acquérir des compétences techniques importantes en informatique.

J’ai notamment amélioré mes notions dans le langage PHP car ce projet avait pour objectif d'appliquer les différents principes que nous avons étudiés en cours.

Par ailleurs, j’ai pu développer mes connaissances en base de données et plus précisément en PostgreSQL. Ce projet fut le premier projet que j’ai réalisé sous PostgreSQL. Un membre de mon groupe voulait nous faire découvrir cet outil de gestion de base de données. Cela nous a permis de nous démarquer des autres projets avec cette prise de risque.

J'ai également ajouté de nombreuses fonctionnalités pour améliorer l'expérience utilisateur. Nous avons donc réalisé une maquette sur Figma, pour la présenter au client. Par la suite, nous l'avons reproduit en ajoutant les différentes demandes de celui-ci.

Par ailleurs, le client attachait une grande importance à la sécurité, c'est pourquoi cela était une priorité pour nous. Nous avons donc mit en place des mesures pour protéger les données et éviter les failles de sécurité.

En outre, nous avons réalisé ce projet en utilisant des méthodes agiles. Nous avons donc organisé des revues de sprint pour présenter au client l'avancée du projet et prendre en compte ses différentes demandes. Par ailleurs, nous avons pu découvrir l'utilisation des outils comme Trello pour organiser les différentes tâches à réaliser à travers le projet.

Ainsi, ce projet nous a permis de découvrir de nouveaux outils et de nous familiariser avec les méthodes Agile notamment SCRUM. De plus, j’ai développé mes compétences techniques en ajoutant des fonctionnalités que nous n’avions pas forcément vu durant les cours. Enfin, cela m’a aidé à me préparer à mon stage car j’ai pu réaliser les différentes demandes d’un client. je serai donc amené à faire de même durant ces 3 prochains mois.

Améliorations

Après avoir travaillé sur ce projet pendant plus de 3 mois, nous avons pu apporter une base solide au projet avec un site entièrement sécurisé et construit avec un modèle MVC. Néanmoins, nous sommes conscients que nous pouvons l'améliorer en apportant de nouvelles fonctionnalités.

Ainsi, on peut aussi développer une application mobile qui reprend les fonctionnalités de l'application Web que nous avons créée. Cela permettra aux utilisateurs de l'utiliser partout où ils se trouvent.

De plus, nous pourrions développer la partie personnalisation pour l'utilisateur avec notamment la mise en place d’une page profil que chacun pourra consulter. Ainsi, que l’ajout de photo de profil et la mise en place de questions épinglées sur la page.
